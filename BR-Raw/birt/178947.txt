Bug 178947 ClassCastException for emitter render option
In current engine code, it assumes all emitter render option is an IRenderOption instance, actually from the api: EngineConfig.setEmitterConfiguration( String format, Object emitterConfig ), it doesn't reflect this requirement, this implicity allows user to user its own render option object type, for example, HashMap, String or other classes.
Actually I think this emitterConfig object is supposed to be passed to concrete emitter directly and interpreted by the concrete emitter only, normally it should not be processed by the engine.
Even if engine need some special logic for built-in emitter like html,pdf, it should always check this instance type before processing.
