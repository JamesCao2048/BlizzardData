Provide build.xml target to export binary plugin.
It's been noted that most Eclipse teams still maintain exportplugin.xml build files for building their plugins locally instead of using PDE's generated build.xml file.
The only real problem that the build.xml file has is that it dumps the output into the input directory.
The nice thing about the exportplugin.xml files is that they build the plugins and output the binary plug-in in a plugin-export directory that is at the same level as the workspace, not inside it.
