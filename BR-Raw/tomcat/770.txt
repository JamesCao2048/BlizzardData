Embedding Excel in IEditorPart changes Workbench menus
Overview/Steps to reproduce : Embed .xls file as OLE document in the IEditorPart.
If that editor is opened, the workbench menus are replaced by the .xls menus {Edit, View, Insert, Format, Tools, Data, Help}.
Even if the active editor is changed to something else, the original workbench menus do not show up.
The only way to get back the workbench menus is to close the editor that has OLE document.
Expected behavior : When the active editor is say Java Editor, the workbench should show the original set of menus, and not the OLE document specific 
menus.
When the active editor is the one embedding the OLE doc, the document specific menus should be shown.
