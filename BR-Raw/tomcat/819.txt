AspectJ does not honour libraries defined via Project Properties
AJDT 1.1.3, Eclipse 2.1 To build a JDK 1.4.program under Eclipse using JDK 1.3 the appropriate build library should be selected in the Project Properties "Java Build Path".
However AspectJ continues to use the JDK associated with Eclipse so errors result from using 1.4 APIs.
The same will occur running Eclipse under 1.4 and compiling programs for 1.5 Testcase: 1.Run Eclipse under JDK 1.3 2.Create a Java project which uses a 1.4 JDK 3.Write a Java classes that uses 1.4 APIs e.g.
public class TestStringBuffer { public static void main(String[] args) { new StringBuffer("Test").indexOf("T"); } } 4.Convert to AspectJ project
