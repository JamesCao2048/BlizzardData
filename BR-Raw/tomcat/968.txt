NPE displaying JavaMethodBreakpoint that has no signature
I created a JavaMethodBreakpoint using JDIDebugModel.createMethodBreakpoint(), specifying null for the methodSignature.
This lead to a NPE when the UI tries to display the breakpoint in the Breakpoints view.
I saw this problem while using the M3 driver.
